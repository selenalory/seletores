Guia prático de seletores CSS

Exemplos visuais para aula introdutória de seletores

1. O problema inicial

Em CSS, o seletor responde à pergunta: em quais elementos esta regra será aplicada?

A ideia desta página é mostrar, de forma simples, como diferentes seletores escolhem elementos diferentes.

2. Seletor universal e seletor por tipo

Exemplo visual

Este parágrafo recebe estilo do seletor por tipo p.

Todos os elementos da página também são afetados pelo seletor universal *, usado aqui em ajustes gerais. 

* {
  box-sizing: border-box;
}

p {
  color: #334155;
  line-height: 1.6;
}

3. Classe e ID

Seletor de classe

Este parágrafo usa a classe .destaque.

Outro elemento pode reutilizar a mesma classe. 

.destaque {
  background: #eff6ff;
  border-left: 4px solid #2563eb;
  font-weight: bold;
}

Seletor de ID

Este parágrafo usa o ID #aviso-unico. 

#aviso-unico {
  color: #8b0000;
  font-style: italic;
}

4. Combinadores

Descendência

  • Início
  • Contato

O seletor nav a atinge os links <a> dentro de <nav>. 

nav a {
  text-decoration: none;
  color: #1d4ed8;
}

Filho direto

  • Item A
  • Item B
    • Subitem B.1
  • Item C

O seletor ul.menu-filho > li pega apenas os li do primeiro nível. 

ul.menu-filho > li {
  background: #f8fafc;
  padding: 6px;
}

6. Pseudo-elementos

::before e ::after

Este texto recebe conteúdo visual antes e depois. 

.aviso-pseudo::before {
  content: ">> ";
  color: #2563eb;
  font-weight: bold;
}

.aviso-pseudo::after {
  content: " <<";
  color: #ea580c;
  font-weight: bold;
}

::first-letter

Este parágrafo destaca visualmente a primeira letra por meio de um pseudo-elemento. 

.frase-pseudo::first-letter {
  font-size: 2rem;
  font-weight: bold;
  color: #0b51bb;
}

7. Conflito entre regras e especificidade

Este texto recebe ao mesmo tempo seletor por tipo, por classe e por ID. 

p {
  color: #334155;
}

.destaque {
  color: #ea580c;
}

#principal {
  color: #b91c1c;
}

Nesse caso, a cor aplicada vence pelo seletor de ID, porque ele tem mais especificidade.

8. Síntese

  • Seletores escolhem elementos.
  • Classe é reutilizável.
  • ID deve ser único.
  • Combinadores olham a estrutura do HTML.
  • Pseudo-classes olham estado ou posição.
  • Pseudo-elementos atuam em partes do elemento ou geram conteúdo visual.
  • Em conflitos, importa a especificidade.
